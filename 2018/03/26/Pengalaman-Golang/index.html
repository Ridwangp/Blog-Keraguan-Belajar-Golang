Pengalaman Belajar Golang

Golang

Setelah hampir 2 bulan belajar mengenai ASP.Net MVC dan Node.js di makers, saya akhirnya mendapat kesempatan untuk belajar bahasa pemrograman yang baru yaitu golang. Menurut saya dibandingakan dengan C# dan Javascript bahasa pemrograman golang dalam stuktur penulisaannya lebih mirip dengan C#. Karena untuk menjalankan suatu fungsi pada golang harus dibuatkan fungsi main. Menurut saya, step yang diperlukan untuk memulai belajar golanglebih banyak dibandingkan dengan C# dan Javascript.

Karena ketika kita akan memulai belajar C# dan Javascript, cukup dengan text-editor seperti visual code sudah cukup, tetapi untuk golang kita harus menginstall golang itu sendiri terlebih dahulu, lalu setelah itu kita harus mengatur path pada golang agar sesuai dengan direktori projek yang dibuat. Menurut saya, di golang itu dituntut untuk menulis dengan benar-benar rapi dan efektif. Karena jika kita mendeklarasikan suatu variabel, dan variabel tersebut tidak digunakan, maka secara otomatis akan terjadi error ketika program di jalankan. Jika kita menggunakan text editor Visual Studio Code, maka kita bisa menggunakan fitur otomatis dari Visual Studio Code untuk membuang hal-hal yang tidak diperlukan pada suatu projek golang. Sebagai contoh, jika kita menginclude suatu package pada projek golang, dan package tersebut tidak digunakan , maka ketika kita save maka secara otomatis package tersebut akan hilang.

Menurut saya, pada javascript penulisan dan pendeklarasian suatu fungsi dan variabel masih lebih mudah, sebagai contoh, jika kita tidak mendeklarasikan suatu variabel dengan nama var, let atau const maka program tersebut masih dapat berjalan,tetapi pada c# dan golang ketika ada satu saja variabel yang tidak kita deklarasikan di awal, maka program tersebut pasti akan error. Jadi menurut saya, jika ingin belajar bahasa pemrograman yang lebih mudah dan “bebas” bisa dipilih javascript. Tetapi jika ingin belajar bahasa pemrograman yang rapi, terstruktur dan efektif bisa dipilih C# dan golang.

Keunggulan dari golang

  1. Mendukung konkurensi di level bahasa dengan pengaplikasian cukup mudah
  2. Mendukung pemrosesan data dengan banyak prosesor dalam waktu yang bersamaan (pararel processing)
  3. Memiliki garbage collector
  4. Proses kompilasi sangat cepat
  5. Bukan bahasa pemrograman yang hirarkial, menjadikan developer tidak perlu ribet memikirkan segmen OOP-nya
  6. Package/modul yang disediakan terbilang lengkap. Karena bahasa ini open source, banyak sekali developer yang juga mengembangkan modul-modul lain yang bisa dimanfaatkan
